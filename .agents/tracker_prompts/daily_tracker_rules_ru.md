# Строгие Правила и Дизайн Токены: Daily Tracker

## 1. Глобальные визуальные ограничения и размеры
- **Фоны Темной Темы**: Основной цвет body `#08141b`. Фон трекера `#0b1a22`. Поповер `#08141b`.
- **Акценты**: Золото (`#c69b50`), Рамки (`var(--square-border) = rgba(255, 255, 255, 0.15)`).
- **Рендер временной шкалы**:
  - Ровно 24 строки, представляющие часы (от 0 до 23).
  - Использовать `display: grid`, конкретно `grid-template-columns: 45px 1fr`.
- **Строки задач и Кубики**:
  - `grid-template-columns: 45px 95px 25px 1fr;`
  - Ровно **6 кубиков на строку**, представляющих 10-минутные интервалы.
  - Размеры кубиков: 15px на 15px. Скругление углов (border-radius): 2px.

## 2. Ограничения Палитры
Откажитесь от генерации случайных RGB строк. Цвета должны браться исключительно из следующих категорий:
- **ФОКУС (FOCUS)**: `#5B84B1`, `#6F9BC4`, `#8DB6D9`, `#A9CCEA`
- **ЗДОРОВЬЕ (HEALTH)**: `#558B7E`, `#6DA396`, `#86BBAF`, `#A5D4C9`
- **ДЕЙСТВИЕ (ACTION)**: `#c69b50`, `#CD6155`, `#E59866`, `#F0B27A`
- **ОТДЫХ (REST)**: `#9B59B6`, `#AF7AC5`, `#C39BD3`, `#D7BDE2`

## 3. Правила DOM и Взаимодействий
1. **Никогда не изменяйте массив кубиков динамически через appendChild, если изначально созданы 144 строгие позиции**. Холст трекера концептуально существует как абстрактные массивы; обновления DOM лишь отражают текущее состояние.
2. **Слушатели событий (Event Listeners)**: Добавьте ОДИН глобальный слушатель событий клика (`click` / `pointer`) на главный контейнер `trackerWrapper`. Делегируйте проверки через `e.target.classList.contains('time_cells_cube')`. Не привязывайте индивидуальные слушатели к каждому кубику, чтобы предотвратить утечки памяти и рассинхронизацию состояния.
3. **Защита пустых ячеек**: Если в часовом блоке нет ни одного видимого элемента `.task-item`, убедитесь, что отображается обертка ввода (`.task-input-wrapper`) для взаимодействия с таймлайном. Скрытые задачи (архивированные / `display: none`) НЕ ДОЛЖНЫ препятствовать рендерингу `.task-input-wrapper`.
4. **Создание промежутков (Drag logic)**: Не используйте нативный Drag & Drop браузера для выделения времени. Создание интервала задач (Span) базируется на 2 кликах (Клик 1 начинает функцию `isDragging`, Клик 2 финализирует `dragState.endIndex`), визуально перехватывая движение мыши через добавление CSS класса `.hover-select` на лету.

## 4. Требования к рендеру всплывающего окна (Popover)
- Обязательно использовать `position: absolute` (НИКОГДА не `fixed`).
- Координаты X/Y должны позиционироваться через `getBoundingClientRect()` от кликнутого элемента `time_cells_cube`.
- При вычислении координат по вертикали ОБЯЗАТЕЛЬНО должен учитываться скролл: `rect.top + window.scrollY - 10`.
- Z-index должен быть не менее 1000.
- Если клик выпадает на "пересечение" нескольких задач, вставьте макет CSS Grid в поповер: `grid-template-columns: 140px 260px;` и добавьте панель `<div class="popover-sidebar">`.
