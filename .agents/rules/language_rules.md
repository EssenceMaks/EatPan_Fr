# Language Requirements

## 1. Code Comments and General Documentation
All inline code comments, docstrings, and newly generated documentation files MUST be written in **Ukrainian** or **English**.
**DO NOT use Russian** in the codebase comments or official project documentation.

## 2. Implementation Plans and Communication
The user prefers to receive all structured analytical documentation, reports, and AI explanations in **Russian**.

When creating or updating specific analytical artifacts for the user such as `implementation_plan.md`, `task.md`, or `walkthrough.md`, you MUST write them in **Russian**.

This applies to descriptions, headers, lists, summaries, and explanations during interaction.
