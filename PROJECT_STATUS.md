# EatPan - Статус проєкту та мета

Цей файл містить опис головних цілей проєкту (Що ми хочемо зробити) та поточний стан їх реалізації (Що вже готово).

## 1. Що ми прагнемо зробити у цьому проєкті

*   **Інфраструктура та HA-архітектура (High Availability)**
    *   Розділення на 4 рівні: Маршрутизація (Route 53), Балансувальник (Cloudflare/Nginx), Обчислювальний бекенд (Node 1 / Node 2), База даних і сховище (Суддя + Supabase).
    *   Реалізація Судді (Arbiter) для передачі "Прапора Майстра" і запобігання Split-Brain (втраті даних при відключеннях).
    *   Кешування через Valkey для мілісекундної віддачі даних та зниження навантаження на базу.
    *   Створення гнучкої моделі бази даних за допомогою PostgreSQL (використання JSONB на ранніх етапах).
*   **Бекенд (Django API)**
    *   Розробка мікросервісного або розподіленого бекенду з REST API (на базі Django REST Framework).
    *   Реалізація повного циклу CRUD (створення, читання, оновлення, видалення) для кулінарних рецептів, категорій та книг.
    *   Інтеграція авторизації та аутентифікації користувачів (багатокористувацький режим) за допомогою Supabase GoTrue.
*   **Фронтенд (EatPan_Fr)**
    *   Створення швидкого модульного клієнтського додатка на базі Vanilla JS / CSS-модулів.
    *   Повна синхронізація UI/UX дизайну (з Figma) в розмітку HTML/CSS, інтерактивні форми.
    *   Налаштування асинхронного зв'язку з Django API, гаряче перемальовування інтерфейсу (Hot-reload) після дій користувача.
*   **Основний функціонал застосунку EatPan**
    *   **Кулінарна книга (Recipe Book):** підтримка багаторівневої ієрархії (Книги -> Групи -> Категорії -> Рецепти). Можливість створення кастомних груп користувача.
    *   **Планувальник меню:** створення персоналізованих планів харчування на тиждень (Meal Planning).
    *   **Віртуальний холодильник (Virtual Fridge):** повноцінне управління наявними запасами на кухні, відстеження строків придатності, нагадування.
    *   **Список покупок (Shopping List):** автоматична генерація списків на основі обраних в меню рецептів та того, чого не вистачає у холодильнику.
    *   **Візуальна аналітика харчування:** дашборди енергетичної цінності, фінансових потоків (витрат на продукти), зв'язку між їжею та настроєм (Food Diary).
*   **Організаційні та тестові задачі**
    *   Налаштування логічної структури Docker-контейнерів для простого локального розгортання та продакшену (мережі, volumes).
    *   Проведення навантажувальних тестів (симуляція відключення електроенергії на локальній Node 1).
    *   Дотримання єдиного стандарту документування (усі Markdown-файли ведуться виключно українською мовою).
