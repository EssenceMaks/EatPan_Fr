# Project Wrap-Up Routine

Цей workflow повинен виконуватися кожного разу, коли ми закінчуємо роботу над фічею, модулем, або просто завершуємо поточну сесію.

## 1. Оновлення документації (Обов'язково)

- **Оновити `docs/tasks.md`:** 
  Знайди відповідні задачі та відзнач їх як виконані `[x]`. Всі технічні завдання або рефакторинг від агента фіксуй у блоці `### AGENT TASKS`. Завжди дотримуйся ієрархії: `### Тема` -> `#### Підтема` (наприклад, `#### Frontend: Craft Space`).
  
- **Оновити `CHANGELOG.md`:** 
  Внеси всі значні структурні, візуальні та логічні зміни до відповідного блоку версії (Added, Changed, Fixed, Removed).

## 2. Логування контролю версій (Git)

- Якщо під час сесії створювалися нові гілки коду (branches) або відбувалися злиття (merge), обов'язково задокументуй це в `CHANGELOG.md` (напр.: "Зміни збережено у гілку `task-board` та злито з `main`").

## 3. Дії з Git (Лише за запитом)

- **НІКОЛИ** не виконуй `git commit`, `git push` або `git merge` автоматично під час Wrap-up без прямого дозволу або прохання розробника.
- Виведи статус git (якщо потрібно) та запитай розробника, чи бажає він зафіксувати ці зміни (commit) або відправити їх на сервер (push).
