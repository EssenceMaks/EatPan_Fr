# EatPan Frontend (EatPan_Fr) 🍳📜

Добро пожаловать в репозиторий клиентской части экосистемы **EatPan**. 
EatPan Frontend — это Single Page Application (SPA), созданное на чистом **Vanilla JavaScript** без использования тяжелых фреймворков. Проект выполнен в уникальном RPG-стиле ("Parchment & Ink" — пергамент и чернила), превращая рутинные задачи (планирование еды, покупка продуктов, выполнение бытовых дел) в увлекательный квест.

---

## 🏗 Архитектура и Стек технологий

Проект гордится своей легковесностью и скоростью. Вместо React, Vue или Angular мы используем самописную компонентную архитектуру:

- **Vanilla JS (ES6+ Modules)**: Нативный JavaScript с использованием современных стандартов.
- **Custom Component Class** (`src/core/Component.js`): Базовый класс для всех UI-компонентов. Обеспечивает реактивность и управляет жизненным циклом (`onMount`, `onDestroy`, `template`, `_render`).
- **Custom Router** (`src/core/Router.js`): Встроенный маршрутизатор для навигации без перезагрузки страницы (History API).
- **Failover API Client** (`src/core/ApiClient.js`): Умный сетевой клиент с паттерном Circuit Breaker. Он автоматически переключается между локальным бэкендом, Cloudflare Workers и Render в случае падения одного из узлов.
- **Vanilla CSS / CSS-in-JS**: Использование единой системы дизайн-токенов (`themes.css`) для поддержки темной/светлой темы и RPG-стилизации.
- **Иконки**: Интегрированы [Lucide Icons](https://lucide.dev/).

## 🗺 Основные модули (Features)

1. **Recipe Book (Книга рецептов)** 📖
   Визуализирована в виде разворота старой книги. Интегрирована с категориями, поддерживает закладки (Prepared / Planned) и легкий (lightweight) формат загрузки с бэкенда для экономии памяти.
2. **Task Board (Доска квестов / Задач)** 📜
   RPG-доска задач с многоуровневой системой. Включает матрицу Эйзенхауэра (Eisenhower Matrix), Timeline (хронологию) и Calendar view.
3. **Meal Planner (Планировщик)** 📅
   Визуальный drag-and-drop планировщик расписания еды на неделю с разделением по приемам пищи (завтрак, обед, ужин).
4. **My Kitchen / Pantry (Кладовая)** 📦
   Ваш личный инвентарь продуктов и готовых блюд с отслеживанием сроков годности. Синхронизируется с книгой рецептов.
5. **Shopping Lists (Списки покупок)** 🛒
   Синхронизированные с Планировщиком списки, которые обновляются в реальном времени (через Event-Driven архитектуру) при добавлении блюд в план.

## 🚀 Как запустить проект (How to run)

Поскольку это чистый Vanilla JS без сложных сборщиков (Webpack/Vite не требуются для разработки), запуск проекта максимально прост.

### 1. Локальная разработка
1. Склонируйте репозиторий.
2. Откройте терминал в корневой папке `EatPan_Fr`.
3. Запустите любой простой статический HTTP-сервер. Например, с помощью Python:
   ```bash
   python -m http.server 6800
   ```
   *(Альтернатива: используйте расширение **Live Server** в VS Code).*
4. Откройте в браузере: [http://localhost:6800](http://localhost:6800)

### 2. Подключение к Бэкенду (API Routing)
Наш `ApiClient.js` умный. Он автоматически пытается подключиться к бэкендам в следующем порядке:
1. Локальный Django API (`http://localhost:6600/api/v1`)
2. Cloudflare Worker API (`https://api.eatpan.com/api/v1`)
3. Render Fallback API (`https://eatpan-back.onrender.com/api/v1`)

> **Важно:** Если вы разрабатываете бэкенд локально, убедитесь, что `EatPan_Back` запущен на порту `6600`. Если локальный сервер недоступен, фронтенд **незаметно переключится** на облачный продакшен API.

## 🌐 Деплой (Production Deployment)

Проект является статическим сайтом (Static Site) и может быть легко развернут на любом хостинге (Cloudflare Pages, GitHub Pages, Vercel, Netlify, Nginx). 

**Главное правило при деплое:**
Поскольку это SPA, сервер должен быть настроен так, чтобы перенаправлять все `404 Not Found` запросы обратно на `index.html`. Это позволит встроенному JS-роутеру корректно обрабатывать прямые ссылки (например, `eatpan.com/recipe/123`).

## 🛠 Правила разработки (Contribution Guidelines)

- **State Management (Управление состоянием):** Состояние хранится локально внутри экземпляров классов `Component`. Для кросс-компонентной синхронизации (например, между Планировщиком и Списком покупок) используются нативные глобальные события:
  ```javascript
  // Отправка события:
  window.dispatchEvent(new CustomEvent('shopping-list-updated'));
  // Прослушивание:
  window.addEventListener('shopping-list-updated', this._onUpdate);
  ```
- **Работа с DOM:** Компоненты полностью перерендеривают свой HTML при вызове `this._render()`. Поэтому привязка событий должна использовать **делегирование** (event delegation), чтобы не терять слушателей:
  ```javascript
  this.element.addEventListener('click', (e) => {
    const btn = e.target.closest('[data-action="save"]');
    if (btn) this.saveData();
  });
  ```
- **CSS и Стилизация:** Избегайте inline-стилей для сложных элементов. Используйте классы и глобальные CSS-переменные из `themes.css` (например, `color: var(--c-ink-primary)`), чтобы поддерживать переключение тем (Light, Dark, Sepia).
