# Керівництво та Архітектурні Правила (RULES)

Цей файл містить набір ключових правил та архітектурних особливостей проекту EatPan, яких слід дотримуватися під час подальшої розробки. Ці правила сформовані на основі виявлених багів та обраних шаблонів проектування.

## 1. Архітектура SPA та Дублювання DOM-елементів (Infinite Scroll)

### Проблема:
SPA маршрутизатор проекту EatPan використовує механізм **циклічної прокрутки (Infinite Scroll)** для головних екранів. В `main.js` функція `initInfiniteScroll()` використовує `cloneNode(true)`, щоб створити копії оригінальних блоків (`.original-block`) навколо поточної видимої області користувача.

Це призводить до того, що **ідентифікатори (`id="col-GET"`, `id="mdTasksList"` тощо) повністю дублюються в DOM-дереві** (що технічно порушує стандарти HTML, але є частиною механіки каруселі).

### Правило:
**НІКОЛИ не використовуйте `document.getElementById()` всередині UI-модулів проекту (включаючи `craft_space.js`) для елементів, які знаходяться всередині циклічних блоків-секцій!** 
`getElementById` завжди поверне перший знайдений клон у прихованій зоні DOM, що призведе до оновлення невидимого UI (елемент "зникає", кнопки не працюють візуально).

**Замість цього:**
- Використовуйте `document.querySelectorAll()` для вибору **всіх копій** (як оригіналу, так і клонів) і оновлюйте їх синхронно за допомогою циклу `.forEach()`.
- *Приклад:*
  ```javascript
  const allContainers = document.querySelectorAll('.kanban-column[data-status="DO"] .kanban-cards-container');
  allContainers.forEach(container => { container.appendChild(card.cloneNode(true)); }); // або innerHTML = ...
  ```

## 2. Логіка HTML5 Drag & Drop

### Проблема:
Нативний механізм Drag & Drop у браузерах має строгі (іноді неочевидні) вимоги. Якщо під час події `dragstart` не записати дані у `dataTransfer`, сучасні браузери (Safari, Chrome) миттєво скасовують операцію.

### Правило:
- **Завжди ініціалізуйте payload `dataTransfer` під час початку `dragstart`**, навіть якщо ви користуєтесь глобальною змінною.
  ```javascript
  e.dataTransfer.setData('text/plain', element.dataset.id); 
  e.dataTransfer.effectAllowed = 'move';
  ```
- **Стережіться зміни прозорості `opacity: 0` у початковому потоці виконання:** Браузер генерує візуального "привида" (drag ghost) з точної копії елемента під час закінчення виконання коллбеку `dragstart`. Якщо ви приховаєте елемент до цього, "привид" буде невидимим. Замість цього завжди використовуйте асинхронний відкладений виклик:
  ```javascript
  setTimeout(() => card.classList.add('dragging'), 0);
  ```

## 3. Типізація Ідентифікаторів (Number vs String)

### Проблема:
Зберігання даних у `localStorage` (та отриманих з `json` API) може призвести до зміни типу ID (наприклад, з `1` на `"1"` або навпаки). Під час пошуку за строгим порівнянням (`===`) скрипт може мовчки відмовити у виконанні.

### Правило:
- **Завжди приводьте обидві сторони ідентифікатора до спільного типу (бажано `String`) перед пошуком у масивах!**
  ```javascript
  const ticket = tickets.find(t => String(t.id) === String(targetId));
  ```
