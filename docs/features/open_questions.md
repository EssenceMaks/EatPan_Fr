# Відкриті питання та дискусії (Open Questions)

Цей файл створено для фіксації ідей, пропозицій та відкритих питань щодо покращення модулів проекту, які зараз знаходяться в процесі обговорення.

## Крафт Спейс (Craft Space / Kanban)

### 1. Проблема відображення статусу кнопок
- **Симптом:** При натисканні на кнопки GET / DO / VERIFY / DONE завдання візуально не перестрибує в іншу колонку. Хоча логіка зміни статусу (`ticket.status`) працює, DOM не перемальовується так, як очікувалось.
- **Можливе рішення:** Замість повного ре-рендерингу всього канбану (`innerHTML = ''`), краще реалізувати пряме переміщення DOM-вузла картки за допомогою `appendChild` у нову колонку. Це буде миттєво і не викличе втрату стану.

### 2. Баг Drag-and-Drop на ПК (скидання "grabbing")
- **Симптом:** При спробі потягнути картку вона зажимається, але при початку руху стан "grabbing" злітає і перетягування скасовується.
- **Причина:** Браузерний API HTML5 вимагає обов'язкової передачі хоча б базового payload у подію старту `event.dataTransfer.setData('text/plain', ...);`. Без цього браузер (особливо Chrome/Edge) миттєво скасовує подію перетягування.
- **Рішення:** Додати `setData` та прибрати конфліктуюче `opacity: 0`.

### 3. Мобільний D&D (Активація довгим тапом - Long Press)
- **Симптом/Ідея:** На мобільних екранах зараз немає нативного Drag-and-Drop. Було запропоновано вимикати звичайний функціонал (наприклад, скрол) і активувати "Режим перенесення картки", якщо користувач робить довге затискання (Long Press) на самій картці. Далі картка починає рухатися за пальцем. Для вимикання цього затискання в меню є подія "Назад" (наприклад, кнопка зліва знизу).
- **План дій:** Замість підключення бібліотеки, можна дійсно написати кастомний Vanilla JS скрипт на 100-200 рядків із `touchstart`, `touchmove`, обробленими через `setTimeout` на 500мс. Якщо натиснуто більше півсекунди — картка стає "відірваною" від списку, `position: fixed`, її координати слідують за пальцем. Якщо палець відпущено, перевіряємо через `elementFromPoint(x, y)`, чи є там `.kanban-column`, і якщо так — перекидаємо картку.
