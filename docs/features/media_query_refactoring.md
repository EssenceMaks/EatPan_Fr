# Media Query Refactoring

## Короткий зміст
Ми повністю переробили систему медіа-запитів, впровадивши структурований 7-рівневий підхід до адаптивного дизайну для всього застосунку. Особливу увагу було приділено Книзі Рецептів: тепер вона гарантовано вписується в екран комп'ютера або планшета без зайвих скролів поза контентом. Залежно від орієнтації та розміру екрана до книги застосовується «вписування по висоті» або «по ширині». Також для мобільних версій було оптимізовано розміри головного меню, щоб максимізувати корисний простір для сторінок.

---

## Правила масштабування Книги Рецептів

Нижче наведено фактично реалізовані правила розміщення Книги Рецептов для різних пристроїв (щоб уникнути появи зовнішнього скрола та зробити читання комфортним):

1. **QHD / 2K та вище (2560x1440 і більше)**
   - Орієнтація: Альбомна (Landscape)
   - Масштабування: **Вписати по висоті** (по ширині вона точно поміститься).

2. **Full HD (1920x1080)**
   - Орієнтація: Альбомна
   - Масштабування: **Вписати по висоті**.

3. **Wide Screen (1440x900)**
   - Роздільна здатність: від 1440px до 1919px
   - Орієнтація: Альбомна
   - Масштабування: **Вписати по висоті**.

4. **Desktop (1280x800)**
   - Роздільна здатність: від 1200px до 1439px
   - Орієнтація: Альбомна
   - Масштабування: **Вписати по висоті**.

5. **Large Tablet Landscape (наприклад, 1366x1024)**
   - Роздільна здатність: 1024px - 1199px (Ландшафтний)
   - Масштабування: **Вписати по висоті**.

6. **Large Tablet Portrait (наприклад, 1024x1366)**
   - Роздільна здатність: 1024px - 1199px (Портретний)
   - Масштабування: **Вписати по ширині** (по висоті місця точно вистачить).

7. **Tablet Landscape (наприклад, 1024x768)**
   - Роздільна здатність: 768px - 1023px (Ландшафтний)
   - Масштабування: **Вписати по висоті**.

8. **Tablet Portrait (наприклад, 768x1024)**
   - Роздільна здатність: 768px - 1023px (Портретний)
   - Масштабування: **Вписати по ширині** (по висоті місця точно вистачить).

9. **Смартфони (Дрібні та Великі)**
   - Орієнтація: Ландшафтна
   - Масштабування: Залишено вписування по ширині з легким масштабуванням.
   - Орієнтація: Портретна
   - Масштабування: Вписувати по ширині (як вже реалізовано `--book-width: 95vw`).

## Детальний опис технічних змін

### 1. Глобальні зміни сітки та меню (global.css)
* **CSS-змінна для висоти меню:** Щоб позбавити застосунок від жорстко заданих розмірів інтерфейсу, у `.layer` замість функції `clamp(25px, 15vh, 100px)` впроваджено змінну `var(--header-height)`.
* **Динамічність шарів:** Це дозволяє змінювати висоту верхнього (`header`) та нижнього (`footer`) меню незалежно одне від одного залежно від поточного пристрою прямо через медіа-запити.
* **Результат для мобільних:** На смартфонах висота меню стала значно меншою (наприклад, `clamp(35px, 10vh, 45px)`), що звільняє місце для критично важливого контенту (самої книги).

### 2. Нова 7-рівнева структура медіа-запитів (media.css)
Увесь старий CSS був переведений на точні брейкпоінти для обох орієнтацій (`portrait` та `landscape`), слідуючи новій матриці пристроїв.

### 3. Вписування Книги Рецептів: Альбомна орієнтація (По висоті)
Найбільшою проблемою на пристроях типу Desktop, Wide Screen та Landscape-планшетах був вертикальний переповнюючий скрол (книга просто йшла вниз за екран).
* **Фіксація висоти:** Ми задали жорстку формулу для висоти контейнера `.book-wrapper: calc(100vh - var(--header-height)*2 - 40px)`. Це гарантує, що книга **фізично не може** вилізти за межі екрана по висоті.
* **Внутрішній скрол замість зовнішнього:** Були скасовані властивості `min-height`, що розтягували обгортку та самі сторінки (`.page { min-height: 0 !important }`). Тепер, якщо контент не поміщається (звузили вікно), вмикається правильний **внутрішній скрол** (властивість `.scrollable-area`).

### 4. Вписування Книги Рецептів: Портретна орієнтація (По ширині)
Для великих вертикальних планшетів (Large Tablet Port та Tablet Port) висота екрана величезна, тому книгу було важливо зробити комфортної ширини.
* **Ширина 93vw:** Книзі задано ширину `width: 93vw` (для великих планшетів) або `95vw` (для смартфонів). 
* **Вільна висота:** При цьому висота встановлена на `auto:`, тому книга просто розкривається вниз і без проблем вміщується на довгому екрані планшета.

### 5. Підтримка надвисоких роздільних здатностей (2K і вище)
У фінальних правках ми видалили обмежувачі максимальної висоти (`max-height: 850px`) для екранів типу Desktop, Wide Screen, 2K/4K (`1440px-1919px` та `1920px+`).
* **Вільне масштабування:** Тепер на великих екранах книга безперешкодно заповнює всю висоту дисплея (від верхнього до нижнього меню), а її ширина (блок `.book-wrapper`) розраховується пропорційно цій висоті (через `calc()`). Це робить книгу великою і зручною на будь-якому величезному моніторі.
